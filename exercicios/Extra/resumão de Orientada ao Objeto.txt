1. Ambiente e Funcionamento do Java
Multiplataforma: O Java é compilado para Byte Code, um formato que não é lido diretamente pelo hardware
.
JVM (Interpretador): A Java Virtual Machine traduz o Byte Code para cada sistema específico. Por isso, cada equipamento precisa de sua própria JVM para rodar programas Java
.
JDK vs JRE: O JRE contém a JVM e bibliotecas para executar programas. O JDK é o pacote para desenvolvedores, contendo o compilador e ferramentas de criação
.
2. Modelagem com UML (Linguagem de Modelagem Unificada)
Padrão de Indústria: A UML é o padrão mundial para modelar sistemas antes da codificação
.
Diagrama de Classe: É dividido em três partes: Nome da Classe, Atributos e Métodos
.
Simbologia de Acesso:
(+) Público: Acesso de qualquer lugar
.
(-) Privado: Acesso apenas pela própria classe
.
(#) Protegido: Acesso pela classe, subclasses e mesmo pacote
.
(~) Default: Acesso apenas dentro do mesmo pacote
.
3. Conceitos Centrais de POO
Classe: É a estrutura que abstrai um conjunto de objetos com características similares (o "molde")
.
Objeto: É a instância real de uma classe
.
Construtor: Método especial com o mesmo nome da classe usado para inicializar o objeto no momento do new
. Pode haver mais de um construtor na mesma classe (sobrecarga), diferenciados pelo número ou tipo de parâmetros
.
4. Atributos, Métodos e Parâmetros
Atributo: Variáveis que representam o estado ou características do objeto
.
Método: Funções que definem o comportamento ou as funcionalidades
.
Retorno: Métodos podem retornar valores (especificando o tipo) ou nada (void)
.
Parâmetros: Valores passados para o método durante a chamada para realizar tarefas específicas
.
5. Encapsulamento e a Palavra-Chave this
Proteção de Dados: Consiste em esconder os atributos (geralmente como private) para garantir que a integridade do objeto não seja violada por acessos externos diretos
.
Getters e Setters: Métodos públicos usados para, respectivamente, recuperar e atribuir valores aos atributos privados de forma controlada
.
Uso do this: Utilizado dentro da classe para referenciar o próprio objeto, sendo essencial para diferenciar um atributo de um parâmetro com o mesmo nome (ex: this.nome = nome;)
.
6. Lógica de Programação e Estruturas de Dados
Tipos de Dados: String (textos), int (inteiros), double/float (reais) e boolean (lógico: true/false)
.
Constantes (final): A palavra-chave final impede que o valor de uma variável seja alterado após a atribuição
.
Vetores e Matrizes:
Vetor: Coleção unidimensional de elementos do mesmo tipo. O índice começa em 0
.
Matriz: Coleção bidimensional (tabela) acessada por dois índices (linha e coluna)
.
7. Controle de Fluxo
Condicionais: if/else para decisões lógicas e switch case para múltiplas opções de escolha fixa
.
Laços de Repetição:
For: Usado quando se sabe o número exato de repetições
.
While: Executa enquanto uma condição for verdadeira; testa a condição antes do bloco
.
Do...While: Executa o bloco pelo menos uma vez antes de testar a condição
.
8. Tratamento de Exceções
O que são: Eventos que interrompem o fluxo normal (erros de execução como divisão por zero ou índice de array inválido)
.
Try/Catch: O código de risco vai no bloco try, e a solução ou mensagem de erro vai no catch
.
9. Reutilização e Conceitos Avançados (Breve Introdução)
Herança: Criação de subclasses que herdam atributos e métodos de uma classe mãe
.
Polimorfismo: Métodos que assumem "muitas formas" (sobrescrita e sobrecarga)
.
Abstração e Interfaces: Definição de moldes que servem apenas como base e não podem ser instanciados diretamente


Exceções Mencionadas:

ArithmeticException: Ocorre em erros de cálculos matemáticos, como a divisão por zero

ArrayIndexOutOfBoundsException: Disparada ao tentar acessar um índice inexistente em um array (ex: acessar a posição 3 em um vetor que só possui índices de 0 a 2)

NumberFormatException: Acontece em falhas de conversão de tipos, como tentar transformar uma String "abc" em um número ou usar vírgula em vez de ponto em um Double.parseDouble()

InputMismatchException: Ocorre quando o dado digitado pelo usuário não condiz com o tipo esperado pelo Scanner (ex: digitar uma letra quando se espera um número inteiro)

Métodos Auxiliares de Captura de exceções
e.getMessage(): Retorna apenas a mensagem de erro específica daquela falha

e.toString(): Converte a exceção em uma representação textual completa, incluindo o tipo da exceção e a mensagem, facilitando a compreensão do erro pelo programador ou usuário

codigos que podem ser uteis na hora de programar:
O método equals() é usado para comparar o conteúdo de dois objetos para verificar se eles são logicamente iguais.   EX:(nome.equals("Ricardo")).

toString(): Este método é utilizado para converter um objeto em uma representação textual
. Nas fontes apresentadas, ele aparece especificamente no contexto de tratamento de exceções, servindo para transformar um erro capturado em uma mensagem legível que descreve os detalhes do problema para o usuário ou programador, serve para passar um numero para String também.
. O comando é aplicado diretamente ao objeto da exceção, como em e.toString()

Double.parseDouble()/float/int: É um método utilizado para converter uma String em um valor numérico do tipo double
. Ele é fundamental quando o programa lê dados do teclado em formato de texto (como "25.5") e precisa transformá-los em números para realizar cálculos matemáticos
. O comando segue a sintaxe Double.parseDouble(variavelString) e pode disparar a exceção NumberFormatException caso o texto contenha caracteres inválidos, como uma vírgula em vez de ponto

Arrays.toString(): faz aparecer todo o vetor  EX:Arrays.toString(nome)



Exemplo pratico de todos os conceitos:

import java.util.Scanner;

class Produto {
    private String nome;
    private double preco;
    private int quantidadeEstoque;

    // Construtor 1: Apenas nome
    public Produto(String nome) {
        this.nome = nome;
        this.preco = 0.0;
        this.quantidadeEstoque = 0;
    }

    // Construtor 2: Nome e Preço
    public Produto(String nome, double preco) {
        this.nome = nome;
        this.preco = preco;
        this.quantidadeEstoque = 0;
    }

    // Getters e Setters
    public String getNome() { return nome; }
    public void setNome(String nome) { this.nome = nome; }

    public double getPreco() { return preco; }
    public void setPreco(double preco) { this.preco = preco; }

    public int getQuantidadeEstoque() { return quantidadeEstoque; }
    public void setQuantidadeEstoque(int quantidadeEstoque) { this.quantidadeEstoque = quantidadeEstoque; }

    public void exibirInformacoes() {
        System.out.println("\n--- Dados do Produto ---");
        System.out.println("Nome: " + nome);
        System.out.println("Preço: R$ " + preco);
        System.out.println("Estoque: " + quantidadeEstoque + " unidades");
    }
}

public class MainProduto {
    public static void main(String[] args) {
        Scanner sc = new Scanner(System.in);
        Produto p = null;
        int opcao;

        do {
            System.out.println("\nMENU");
            System.out.println("1 - Criar produto informando nome");
            System.out.println("2 - Criar produto informando nome e preço");
            System.out.println("3 - Exibir informações do produto");
            System.out.println("4 - Atribuir nome do produto");
            System.out.println("5 - Obter nome do produto");
            System.out.println("6 - Atribuir preço do produto");
            System.out.println("7 - Obter preço do produto");
            System.out.println("8 - Atribuir quantidade em estoque");
            System.out.println("9 - Obter quantidade em estoque");
            System.out.println("0 - Sair");
            System.out.print("Escolha: ");
            opcao = sc.nextInt();
            sc.nextLine(); // Limpar buffer

            switch (opcao) {
                case 1:
                    System.out.print("Nome: ");
                    p = new Produto(sc.nextLine());
                    break;
                case 2:
                    System.out.print("Nome: ");
                    String n = sc.nextLine();
                    System.out.print("Preço: ");
                    double pr = sc.nextDouble();
                    p = new Produto(n, pr);
                    break;
                case 3:
                    if (p != null) p.exibirInformacoes();
                    else System.out.println("Produto não criado!");
                    break;
                case 4:
                    if (p != null) { System.out.print("Novo nome: "); p.setNome(sc.nextLine()); }
                    break;
                case 5:
                    if (p != null) System.out.println("Nome: " + p.getNome());
                    break;
                case 6:
                    if (p != null) { System.out.print("Novo preço: "); p.setPreco(sc.nextDouble()); }
                    break;
                case 7:
                    if (p != null) System.out.println("Preço: " + p.getPreco());
                    break;
                case 8:
                    if (p != null) { System.out.print("Nova qtd: "); p.setQuantidadeEstoque(sc.nextInt()); }
                    break;
                case 9:
                    if (p != null) System.out.println("Qtd: " + p.getQuantidadeEstoque());
                    break;
            }
        } while (opcao != 0);
    }
}

abstract class Funcionario {
    protected String nome;
    protected double salarioBase;

    public Funcionario(String nome, double salarioBase) {
        this.nome = nome;
        this.salarioBase = salarioBase;
    }

    public void exibirDados() {
        System.out.println("Funcionário: " + nome + " | Salário Base: R$ " + salarioBase);
    }

    public abstract double calcularSalarioFinal();
}

class FuncionarioCLT extends Funcionario {
    public FuncionarioCLT(String nome, double salarioBase) {
        super(nome, salarioBase);
    }

    @Override
    public double calcularSalarioFinal() {
        return salarioBase + 500.00;
    }
}

class FuncionarioPJ extends Funcionario {
    public FuncionarioPJ(String nome, double salarioBase) {
        super(nome, salarioBase);
    }

    @Override
    public double calcularSalarioFinal() {
        return salarioBase;
    }
}

interface Controlavel {
    void ligar();
    void desligar();
}

class Televisao implements Controlavel {
    private int canalInicial = 5;

    public void ligar() {
        System.out.println("TV Ligada. Canal inicial: " + canalInicial);
    }

    public void desligar() {
        System.out.println("TV Desligada.");
    }
}

class Lampada implements Controlavel {
    private boolean ligada = false;

    public void ligar() {
        ligada = true;
        System.out.println("Lâmpada LIGADA (Estado: " + ligada + ")");
    }

    public void desligar() {
        ligada = false;
        System.out.println("Lâmpada DESLIGADA (Estado: " + ligada + ")");
    }
}

        lamp.desligar();
    }
}
